# GOKU SoDEX Operator

Wave 3 submission for **Build Your One-Person On-Chain Finance Business with SoSoValue**.

GOKU SoDEX Operator is a real trading workstation for a solo builder who wants to:

- read live SoDEX spot structure
- consume live SoSoValue research and macro context
- compare ideas against smart-money positioning
- stage auditable execution drafts
- prepare signed SoDEX orders before live submit

This repo is intentionally opinionated: if a module is visible in the UI, it should map to a real operator workflow, a real exchange endpoint, or a real research payload.

## Product Thesis

Most hackathon trading tools stop at one of two layers:

- good research, weak execution
- flashy execution, weak thesis

GOKU joins both.

The product acts like an **operator rail** for one-person on-chain finance:

1. detect opportunity from SoDEX live tape
2. validate thesis with SoSoValue news and macro context
3. compare against smart-money consensus from SoDEX leaderboard positioning
4. run risk gates before action
5. generate and store execution drafts
6. prepare the exact signed SoDEX payload for live submission

## Live Modules

### `Launch`

- loads live SoDEX spot tickers and book tickers
- computes breadth and regime verdict from current market state
- shows SoSoValue research trigger from live news payloads

### `Strategy Rack`

- turns live SoDEX rows into staged execution ideas
- ships three actionable rails:
  - Trend Capture
  - Mean Reversion
  - Vol Breakout
- stores drafts and rationale into local audit storage

### `Replay Lab`

- replays live SoDEX klines
- scores strategy behavior with return, Sharpe, drawdown, and win rate

### `Smart Money Mirror`

- reads SoDEX leaderboard
- inspects live perps positions from top wallets
- builds consensus by symbol, side, and notional dominance
- supports custom peer-wallet tracking and mini scorecards

### `LP Guard`

- reads live SoDEX orderbook
- calculates spread, bid depth, ask depth, and imbalance
- suggests the maker side from current book pressure

### `Execution Copilot`

- resolves SoDEX symbol metadata
- probes fee visibility
- checks `account_id`, min notional, and max notional gates
- merges:
  - live tape
  - SoSoValue symbol research
  - SoDEX smart-money consensus
- prepares a signed SoDEX payload for review
- can attempt live submit when the wallet and account are properly configured

### `News Intelligence`

- reads live SoSoValue hot news
- reads live SoSoValue featured news
- supports AI summarization into action drafts through Groq

### `Operator Queue`

- keeps execution drafts generated by strategy, news, or copilot modules

### `Portfolio Live`

- reads live SoDEX balances
- reads live SoDEX account state
- reads live SoDEX open orders

### `Audit Trail`

- stores recent drafts and decisions in SQLite
- keeps the operator workflow reviewable and submission-friendly

### `Diagnostics`

- verifies SoDEX and SoSoValue connectivity
- surfaces latency and config readiness

## Builder Differentiation

This project is not a generic market dashboard.

It borrows discipline from strong open-source trading systems and prediction-market builders, then re-centers the stack on **SoDEX execution + SoSoValue intelligence**:

- backtesting mindset from replay-first systems
- smart-money mirroring from trader behavior tools
- execution hygiene from order-management bots
- typed AI drafting from agent-oriented products

The result is a single builder desk focused on one question:

**"Given this live SoDEX market and this live SoSoValue context, should I act, size down, or skip?"**

## Demo Flow For Judges

Recommended flow during judging:

1. Open `Launch`
   - show breadth, regime verdict, and live research trigger
2. Open `Smart Money Mirror`
   - show which symbols top wallets are actually leaning into
3. Open `LP Guard`

4. Open `Execution Copilot`
   - show:
     - live symbol metadata
     - fee probe
     - smart-money verdict

     - signed order preparation

## Important Constraints

This repo is honest about the current boundary:

- market data, book data, symbol metadata, leaderboard, and research can all run without personal account activity
- personal portfolio and live order history require a real SoDEX wallet with an active account
- if the configured wallet resolves to `account_id = 0` or has no active state, the app will still function as a live market and execution-prep terminal, but portfolio surfaces will remain sparse
